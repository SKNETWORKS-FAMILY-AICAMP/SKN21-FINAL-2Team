# **Triver : LLM 기반 대화형 여행 상품 추천 서비스**

## **1. 프로젝트 주제**

**당신이 원하는 한국 여행의 모든 것, 초개인화 여행 에이전트**

- 외국인 관광객들의 국내 여행 전 과정을 책임지는 LLM 기반 AI 여행 에이전트. 단순히 정보를 나열하는 기존 서비스와 달리, 사용자의 취향과 대화 맥락에 대한 이해를 바탕으로 여정을 설계함. 멀티모달 Vision 기술을 통해 사진 한 장으로 숨은 명소와 그 속에 담긴 문화적 맥락을 즉각 찾아내며, RAG 기술을 접목해 실시간으로 검증된 정확한 정보만을 제공함. 특히 LLM의 추론 능력에 수학적 최적화 알고리즘을 결합하여, 실시간 교통량과 영업 상태가 반영된 '실행 가능한 타임라인'을 자동으로 생성함. 사용자는 복잡한 계획 수립을 AI에 위임하고, 에이전트가 제시하는 최적의 플랜을 따라 온전히 여행을 누리는 데 집중할 수 있게 됨.

## **2. 문제 정의 (Problem Definition)**

### **2.1 기획 배경**

- **문화콘텐츠의 확산과 한국 관광 수요 증가**
    - 2025년 실시된 국가 이미지 조사에 따르면, 외국인의 82.3%가 한국을 긍정적으로 평가하고 있으며, 이는 조사가 시작된 2018년(78.7%) 이래 가장 높은 수치로 확인됨. 이러한 호감도의 근간에는 K-pop의 확산과 같은 문화콘텐츠의 역할이 8할을 차지함.
    - 드라마와 영화를 통해 노출된 한국의 현대적 라이프스타일과 제품/브랜드에 대한 선호는 자연스럽게 해당 문화를 직접 체험하고자 하는 방한 욕구로 이어지고 있음.
    - 이러한 욕구에 따라 관광객들의 관광 형태도 과거의 정형화된 단체 관광에서 벗어나, 개인의 성향에 맞춘 개별 자유 여행 형태로 변화함.
- **국내 특화적이고 파편화된 여행 정보 서비스**
    - 위와 같은 패러다임의 전환은 한국 관광 산업에 전례 없는 기회를 제공하는 동시에, 디지털 장벽, 제도적 폐쇄성, 그리고 로컬 인프라의 미비점이라는 과제를 수면 위로 끌어올리기도 했음.
    - 세계적인 지도 어플리케이션 **구글맵의 무력화**는 관광객들의 국내 여행지 탐색에 대한 높은 진입 장벽을 형성함. 국내에서 널리 사용되는 네이버·카카오 지도 서비스는 상호명 및 주소의 번역이 투박하거나 아예 번역되지 않는 등 로컬에 특화되어 외국인이 사용하기에 불편함이 있음.
    - 외국인 관광객들이 한국 여행 중 겪는 고충은 단순한 소통의 어려움을 넘어, 한국 사회 특유의 디지털 환경과 제도적 폐쇄성에서 기인함. 2023년 조사에 따르면 관광객의 58%가 '정보 부족'을 주요 불편 사항으로 꼽았는데, 이는 정보의 절대량이 부족해서가 아니라 '신뢰할 수 있는 다국어 정보'와 '현지 앱 접근성'의 문제로 분석됨.
    - 여행 시작을 위해 언어적 장벽을 넘는 것은 물론, 취향에 맞는 정보를 하나하나 모아 계획을 구성해야 하고, 여행 중에도 간편한 정보 탐색이 어려워 제약이 생긴다면 관광객의 여행 피로도는 더욱 높아질 수밖에 없음. 이는 여행지에 대한 부정적 인식으로 이어져 재방문의 가능성을 낮추고, 관광 수요의 감소로 이어질 수 있음.
- **검색 기능의 한계성**
    - 일반 검색 서비스는 정보를 개별적으로 반환하여, 사용자가 검색된 상품을 직접 탐색하고 비교해야 하는 수고가 발생함.
        
        사용자의 input 텍스트에서 드러나는 맥락이나 실제 검색 의도, 사용자의 현재 상태 및 선호도를 명확히 파악하여 개인화된 상품을 추천해주지는 않음.
        
    - LLM 서비스는 정보를 모아 조합하고 요약해서 전달해주는 역할을 수행하지만, 실제 원본 데이터를 정확하게 가져오거나 참조하지 못해 거짓된 정보를 제공할 위험이 있음.

### **2.2 목표**

- **파편화된 여정의 통합, 원스톱 서비스 : 공신력 있는 공공 데이터의 정확성과 민간 예약 플랫폼의 편의성을 LLM으로 통합하여, 탐색부터 예약까지 이탈 없는 하나의 플로우를 구축.RAG(Retrieval-Augmented Generation)를 통해 검증된 정보를 제공하고, API 연동을 통해 즉각적인 예약이 가능한 대화형 커머스 환경 제공. 단순히 장소 정보를 나열하거나 추천하는 수준을 넘어, 추천된 상품과 장소들을 유기적으로 결합하여 사용자가 즉시 실행할 수 있는 완성된 플랜을 수립해 제공함으로써 사용자는 에이전트가 제시한 계획표를 따라가기만 하면 되는 환경을 구축.**
- **여행 계획 의사결정 피로도 해소 : 서비스 가입 단계에 사용자의 여행 스타일 및 취향에 대한 설문조사를 통해 사용자 맞춤 인터페이스를 사전 구축하고, 이러한 인터페이스 위에서 AI가 사용자의 취향에 맞춰 핵심 정보만 큐레이션함으로써 결정 장애를 해소. 방대한 사용자 리뷰와 파편화된 정보를 사용자가 직접 분석하게 하는 대신, 다수의 리뷰 데이터를 분석·요약하여 장소의 특징, 분위기, 주의사항 등을 맞춤형으로 제안하는 ‘지능형 요 약 에이전트’ 기능 탑재.**
- **동선 최적화(Dynamic Optimization)’ 엔진 구축: 사용자가 고민할 필요 없도록 이동 수단, 예상 소요 시간, 장소 간 최적 순서, 식사 및 휴식 시간까지 포함된 정교한 타임라인을 에이전트가 대신 설계. 각 장소의 실시간 영업 상태, 예약 확정 시간, 이동 거리 등 복잡한 변수들을 알고리즘이 자동으로 계산하여, 여행의 '준비와 설계'라는 번거로운 과정을 에이전트에게 위임함으로써, 사용자가 계획에 드는 에너지를 온전히 '여행의 즐거움과 경험'에만 쏟을 수 있게 함.**
- **다국어 및 멀티모달 기반의 자유로운 탐색 환경 제공: 언어 장벽과 텍스트 검색의 한계를 극복하기 위해 이미지 기반 검색과 고도화된 다국어 지원을 통해 외국인 관광객에게 특화된 접근성을 확보함. Vision 모델을 활용한 이미지 인식 검색과 현지 문화 맥락이 담긴 스토리텔링을 제공하여 정보 접근성과 여행의 깊이를 동시에 충족시킴. 또한, 국내 로컬 지도 앱에서 제공되는 장소별 소비자 리뷰를 다국어로 지원함으로써 실제 로컬 이용자들의 후기를 직접 확인할 수 있음.**

## **3. 기대효과 (Expected Effects)**

### **3.1 사용자 측면**

- **여행 설계 부담 완화:** 여행 전 많은 시간이 소요되던 정보 탐색과 일정 짜기 과정을 에이전트에게 완전히 위임함으로써 사용자는 계획 수립에 대한 부담을 덜 수 있음. 사용자의 현재 위치와 취향 정보를 바탕으로 즉흥적인 이동 계획 또한 수립이 가능하며 사용자 상태에 맞는 실시간 동선 변경도 가능함.
- **초개인화된 K-Culture 몰입 경험 제공: 개인의 취향과 이미지 검색을 통해 발굴된 '나만의 명소'를 방문함으로써, 흔한 관광지 투어가 아닌 본인의 관심사와 긴밀히 연결된 깊이 있는 여행 경험이 가능해짐.**

### **3.2 산업 및 경제적 측면**

- **관광 진입 장벽 완화 및 재방문율 제고: 국내 특유의 폐쇄적인 여행 인프라 문제를 기술적으로 해결하여 외국인 관광객의 접근성을 높이고, 긍정적인 여행 경험을 확산시켜 한국을 '다시 찾고 싶은 국가'로 각인시킬 수 있음.**
- **관광 수요의 다변화와 지역 경제 활성화: 서비스 지원 지역을 수도권에서 전국으로 확대할 경우, 대중적인 관광지에만 쏠리던 인파를 사용자의 세분화된 취향에 맞춰 전국의 '숨은 로컬 명소'로 분산시켜 지역 관광 균형 발전에 기여할 수 있음.**
- **고부가가치 데이터 자산 확보: 외국인 관광객의 실시간 검색 의도, 선호하는 문화 콘텐츠, 이동 패턴 및 소비 데이터를 축적하여, 향후 더 정교한 관광 정책 수립 및 마케팅의 기초 자료로 활용 가능.**
- **K-컬처 콘텐츠의 오프라인 확장성 강화: 미디어 콘텐츠(드라마, 영화 등)에 대한 관심이 일시적인 소비에 그치지 않고, 실제 방문과 소비로 이어지는 선순환 구조를 구축하여 문화 콘텐츠 산업의 경제적 부가가치 극대화**

## **4. 시장조사 및 BM (Market Research & BM)**

### **4.1 현존 서비스 종류 및 분석**

- **정부 공식 여행 정보 플랫폼 “VisitKorea” - 국내**
    - **한국관광공사가 운영하는 만큼 정보의 공신력과 정확성이 매우 높으며, 전국 단위의 방대한 관광지 DB를 다국어로 제공함. 특히 축제 일정, 문화재 개방 시간 등 민간 앱에서 놓치기 쉬운 세밀한 로컬 정보와 정부 인증 숙박/식당 정보를 얻기에 최적화되어 있음.**
    - **정보 제공이라는 공공의 목적에 집중하다 보니 실제 예약 및 결제 기능은 외부 링크 연결에 그치는 경우가 많아 서비스 일관성이 부족함. 사용자의 취향을 학습해 제안하기보다는 백과사전식 나열에 가깝고, 앱의 기술적 안정성이나 사용자 인터페이스(UI) 측면에서 최신 트렌드를 반영한 민첩한 대응이 다소 미흡함.**
- **여행 상품 예약 플랫폼 “마이리얼트립(Myrealtrip)” (국내)**
    - **국내 최대 규모의 투어 및 액티비티 인벤토리를 보유하고 있어, 항공권부터 숙박, 가이드 투어까지 원스톱 예약이 가능하다는 것이 큰 강점임. 특히 현지 가이드가 제공하는 생생한 상품 설명과 검증된 후기는 사용자에게 높은 신뢰도를 제공함.**
    - **예약 효율성에 집중된 커머스 중심 구조이다 보니, 상품 결제 전 단계인 '여정 설계 및 테마 구축' 과정에서는 여전히 외부 채널(SNS, 블로그)을 통한 파편화된 탐색이 동반됨. 특히 외국인 관광객 입장에서는 K-컬처와 관련된 깊이 있는 추천보다 단순 상품 정보 위주의 나열로 느껴져 몰입감이 저하될 수 있음.**
- **AI 기반 여행 및 각종 여행상품 추천 서비스 “트리플(Triple)” (국내)**
    - **'장소 선택 후 동선 자동 계산'이라는 혁신적인 UX를 통해 여행 계획의 번거로움을 획기적으로 낮췄으며, 친구와 일정 공유 및 가계부 기능 등 동행자와의 협업 플래닝에 효과적인 도구를 제공함. 지도를 기반으로 한 직관적인 인터페이스에 국내 여행자들이 높은 만족도를 보임.**
    - **추천 알고리즘이 대중적인 데이터(인기 순위)에 기반하므로, 세밀하고 독특한 개인의 취향을 반영하기에는 추천 고착화 현상이 나타남. 대화형 인터페이스가 아니기에 사용자 개인 상황에 맞는 실시간 추천 조건 업데이트 및 반영이 어렵고, 내국인 위주 서비스 설계로 인해 방한 외국인들에게는 진입 장벽이 존재함.**
- **여행 상품 및 액티비티 예약 서비스 “클룩(Klook)” (글로벌)**
    - 아시아 지역 여행 상품 및 교통권 예약의 글로벌 강자로, 압도적인 가격 경쟁력과 즉시 확정 시스템이 큰 강점임. 입장권, 레일 패스, 유심 등을 한 곳에서 관리할 수 있는 통합 바우처 시스템은 외국인 관광객들에게 높은 신뢰도와 예약 편의성을 제공함.
    - 개별 상품의 판매에는 특화되어 있으나, 이 상품들을 사용자의 특정 테마에 맞춰 유기적으로 연결해 주는 플래닝 도구로서의 역할은 미비함. LLM 기반 서비스와 달리 "K-드라마 촬영지 위주로 효율적인 동선을 짜줘"와 같은 맥락 중심의 요청을 수행하지 못하며, 단순히 나열된 상품 리스트 중 사용자가 직접 자신의 루트에 맞는 정보를 선별해야 하는 번거로움이 있음.
- **여행 리뷰 및 정보 서비스 “트립어드바이저(TripAdvisor)” (글로벌)**
    - 전 세계 수억 건의 실제 방문 후기를 보유한 집단지성의 보고로, 장소의 실제 위생 상태나 친절도 등 생생한 소비자 경험 데이터를 얻기에 최적임. 특히 글로벌 커뮤니티가 활성화되어 있어 외국인 입장에서 느낀 한국 여행지의 장단점을 파악하기에 매우 유용함.
    - 정보의 절대량이 너무 방대하여 사용자가 본인에게 맞는 정보를 선별하기 위해 수많은 리뷰를 직접 정독해야 하는 의사결정 피로도가 큼. LLM 기반 서비스가 수천 개의 리뷰를 즉시 요약하여 사용자 취향에 맞는 결론을 도출해 주는 것과 대조적으로, 여전히 수동적인 검색 방식에 머물러 있어 실시간 상황 변화에 따른 개인화된 추천이나 능동적인 일정 수정 능력은 부족함.

### **4.2 서비스 차별점 및 벤치마킹 포인트**

- **벤치마킹 포인트 (Benchmarking Strategy)**
    - **VisitKorea가 보유한 한국관광공사의 검증된 다국어 관광 데이터베이스를 RAG(검색 증강 생성)의 핵심 소스로 활용. 공공 데이터 특유의 높은 신뢰도를 확보함으로써 LLM의 고질적인 문제인 할루시네이션(정보 왜곡)을 원천적으로 방지.**
    - **클룩과 마이리얼트립의 강력한 장점인 즉시 확정 예약 시스템과 방대한 상품 인벤토리를 벤치마킹. AI가 설계한 일정 내에서 별도의 앱 이탈 없이 예약까지 이어지는 커머스 환경을 구축하여 사용자의 편의성을 극대화.**
    - **트리플이 보여준 직관적인 지도 기반 일정 시각화 UX를 계승하여, 에이전트가 제안한 복잡한 타임라인을 사용자가 한눈에 파악하고 바로 실행에 옮길 수 있는 인터페이스를 제공.**
- **서비스 차별점 (Service Differentiation)**
    - **단순 검색과 나열 방식에서 벗어나 맥락 기반의 자동화된 여정 설계를 제공. 키워드 검색에 의존해야 했던 기존 플랫폼과 달리, 멀티모달(이미지 및 대화) 기능을 통해 사용자가 SNS 사진 한 장만 업로드해도 그 이면의 K-Culture 맥락까지 파악하여 맞춤형 장소를 제안할 수 있음.**
    - **사용자가 수많은 리뷰를 직접 정독하며 판단해야 했던 트리플이나 트립어드바이저의 불편함을 지능형 요약 에이전트 해결. LLM이 수천 건의 리뷰를 실시간으로 분석하여 사용자의 취향에 꼭 맞는 핵심 정보와 주의사항만을 도출해 전달함으로써 의사결정 피로도를 획기적으로 완화.**
    - **동선 수립 단계에서 사용자의 수동 개입을 요구하던 기존 방식과 달리, 이동 수단, 영업시간, 식사 및 휴식 시간을 모두 고려해 자동으로 여정을 설계하여 사용자는 제공된 계획을 실행하기만 하면 됨. 여행 중 갑작스러운 변수가 발생하더라도 현재의 상황에 맞춰 빠르게 새로운 여정을 계획함으로써 돌발 상황에 대한 여행객의 부담을 줄임.**
- **전략적 포지셔닝**
    - **사용자의 조작을 기다리는 단순한 '도구(Tool)'가 아니라, 사용자의 의도를 선제적으로 파악하고 완성된 해결책을 제시하는 '지능형 여행 비서'로 포지셔닝. 국내 지도 서비스의 접근성 한계와 다국어 정보의 부재라는 고질적인 문제를 기술적으로 해결함으로써, 방한 외국인 관광객들에게 한국 여행의 시작과 끝을 책임지는 비서가 되도록 설계.**

### **4.3 수익 모델 및 비즈니스 전략 (Business Model)**

- **예약 수수료 기반의 제휴 커머스(Affiliate Model)**
    - AI가 설계한 여정 내에 포함된 입장권, 액티비티, 숙박 상품을 제휴 플랫폼(Klook, 마이리얼트립 등)과 연결하여 결제 발생 시 일정 비율의 수수료 수익을 창출함.
- **로컬 상권 큐레이션 및 B2B 마케팅**
    - 대중적인 관광지에만 쏠리던 관광객을 전국의 '숨은 로컬 명소'로 분산시키는 대가로, 해당 지자체나 로컬 브랜드로부터 데이터 기반의 홍보 대행 수수료를 수취함.
- **고부가가치 관광 데이터 리포트 판매**
    - 외국인 관광객의 실시간 검색 의도, 선호하는 문화 콘텐츠, 실제 이동 패턴 등의 데이터를 비식별화하여 수집하고, 이를 필요로 하는 관광 정책 수립 기관이나 마케팅 기업에 분석 리포트로 제공함.

## **5. 주요 기능 정의**

### **5.1 맥락 인지형 초개인화 추천 엔진 (Context-Aware Recommendation)**

사용자의 명시적인 요청뿐만 아니라, 이미지, 과거 취향, 실시간 상황을 복합적으로 분석하여 '나보다 나를 더 잘 아는' 여행지를 제안함.

**① 멀티모달 기반 '스냅 탐색' (Visual Search to Place)**

- 설명: 사용자가 캡처한 사진을 업로드하면 Vision AI가 장소를 식별함. 모든 장소를 인식하기보다 K-컬처 주요 명소(촬영지, 랜드마크 등) 데이터셋을 우선적으로 매칭하여 정확도를 높임.
- 상세: 단순 위치 정보 제공을 넘어 해당 장소와 관련된 드라마/영화의 주요 장면 및 관련 에피소드를 LLM이 큐레이션하여 문화적 맥락을 제공함.

**② 대화형 취향 큐레이션 및 리뷰 요약**

- 설명: LLM이 대화를 통해 사용자의 추상적인 요구(예: "BTS가 갔던 고즈넉한 곳")를 파악함.
- 상세: 수천 개의 리뷰를 직접 읽는 피로도를 줄이기 위해, RAG 기반으로 핵심 리뷰 데이터만 추출 및 요약하여 장소의 분위기나 대기 시간 등 의사결정에 필요한 실질적 인사이트를 요약 제공함.

**③ 테마 기반 페르소나 매칭 (Pre-defined Persona Matching)**

- 설명: 서비스 진입 시 진행한 설문조사(여행 스타일, 선호 콘텐츠 등)를 바탕으로 사용자에게 최적화된 인터페이스와 추천 로직을 사전 설정함.
- 상세: '미식가형', 'K-POP 팬덤형', '로컬 탐험가형' 등 사용자의 페르소나에 맞춰 추천 장소의 우선순위가 조정됨. 팬덤형 사용자에게는 아티스트의 발자취가 우선 노출되고, 로컬 탐험가형에게는 잘 알려지지 않은 골목 상권 위주의 정보가 우선적으로 큐레이션될 수 있음.

**④ RAG 기반의 실시간 정보 검증**

- **설명: 추천되는 모든 장소 정보는 공공 데이터 DB와 실시간 장소 API로 대조하여 검증됨.**
- **상세: AI가 존재하지 않는 장소를 지어내는 할루시네이션을 방지하며, 현재 영업 중인지, 해당 날짜에 축제나 행사가 있는지 등의 실시간 유효성을 체크하여 추천 리스트의 신뢰도를 보장함.**

### **5.2 자율 여정 설계 및 동선 최적화 (Autonomous Route Optimization)**

사용자가 선택한 장소들과 AI가 추천한 명소들을 결합하여, 실제 실행이 가능한 최적의 시간표를 자동으로 생성하고 관리함.

**① 알고리즘 기반 '원클릭 타임라인' 자동 생성**

- 설명: 사용자가 가고 싶은 장소들을 확정하면, VRP(Vehicle Routing Problem) 알고리즘을 활용해 이동 거리와 시간을 최소화한 최적의 순서를 산출
- 상세: 단순히 장소 나열에 그치지 않고, 각 장소의 실시간 영업시간, 예상 체류 시간, 이동 수단별 소요 시간을 계산하여 식사 시간과 휴식 시간이 포함된 타임라인을 제공.

**② 국내 지형 특화 '심리스 내비게이션' 가이드**

- 설명: 구글맵 사용이 어려운 한국의 특수한 상황을 고려하여, 에이전트가 국내 지도 API(네이버/카카오 등)를 기반으로 한 상세 이동 경로를 다국어로 안내함.
- 상세: 단순히 "어디로 가라"는 텍스트를 넘어, 지하철 몇 번 출구로 나와야 하는지, 어떤 버스를 타야 하는지 등의 정보를 시각화된 경로와 함께 제공하여 외국인 관광객의 '길 찾기 공포'를 원천적으로 제거함.

**③ 예약 상태 기반의 '스케줄 동기화'**

- 설명: 서비스 내에서 결제된 여행 상품(Klook, 마이리얼트립 등 연동)이나 식당 예약 시간과 연동되어 일정이 고정됨.
- 상세: 예약이 확정된 시간(예: 오후 2시 뮤지컬 입실)을 '고정 노드'로 설정하고, 그 전후 시간을 활용해 방문 가능한 주변 명소를 배치함.

### **5.3 통합 예약 연동 및 바우처 관리 (Integrated Booking & Management)**

직접 결제 시스템을 구축하는 대신, 검증된 글로벌/국내 OTA(Online Travel Agency)의 상품을 AI가 연결해주고 사용자는 한곳에서 관리만 하는 '하이브리드 결제 지원' 모델을 지원함.

**① 여정 기반 '원클릭 예약 링크' 큐레이션**

- 설명: AI가 설계한 일정표 내에 유료 입장권, 교통 패스, 투어 상품이 포함된 경우, 해당 상품을 가장 저렴하게 살 수 있는 제휴사(클룩, 마이리얼트립 등)의 직결 예약 페이지를 버튼 형태로 배치.
- 상세: 직접 결제 모듈을 붙이는 대신 Deep-link(딥링크) 기술을 활용. 사용자가 버튼을 누르면 해당 앱의 결제 화면으로 바로 연결되어, 보안이나 환불 이슈를 제휴사가 책임지게 함으로써 서비스 운영 리스크를 최소화함.

**② 통합 모바일 바우처 지갑 (Digital Pass Wallet)**

- 설명: 여러 플랫폼에서 예약한 다양한 바우처(QR 코드, 예약 번호 등)를 이메일이나 스크린샷에서 찾을 필요 없이, 서비스 내 '마이 여정' 탭 한곳에 자동으로 모아줍니다.
- 상세: 사용자가 예약 확정 메일을 서비스 전용 메일로 포워딩하거나 스크린샷을 업로드하면, OCR(광학 문자 인식) 기술로 정보를 추출하여 타임라인의 해당 시간대에 자동으로 매칭.

**③ 실시간 예약 현황 기반 '스케줄 락(Lock)'**

- 설명: 사용자의 수동 인터랙션을 통해 예약 확정 항목을 체크하고 타임라인에서 '고정된 일정'으로 표시함. AI가 이를 기준으로 동선을 설계 및 조정함.

## **6. 아키텍처 및 모델링 방안**

### **6.1 시스템 아키텍처 (System Architecture)**

장소 검색 > 리뷰 요약 > 일정 생성으로 이어지는 복잡한 논리 단계를 유기적으로 연결하는 에이전틱 워크플로우(Agentic Workflow) 구조를 채택함. LLM은 대화/의도 파악 용도로 활용하고, LLM이 계산하기 어려운 '최단 거리'와 '영업시간 제약'을 전용 알고리즘과 DB(RAG/VRP)에게 맡겨 수학적으로 해결함. 비동기 처리를 통해 사용자에게 즉각적인 인터랙션을 제공하는 구조로 설계함으로써 이같은 분리된 구조로 일반 챗봇과의 차별점 형성하고자 함.

- 인터페이스 레이어 (Client)
    - 사용자의 대화 및 이미지 입력을 수신하고, 최종적으로 시각화된 타임라인을 출력함.
- 오케스트레이션 레이어 (AI Agent)
    - 사용자의 의도를 분석하여 '장소 검색', '리뷰 요약', '동선 최적화' 등 필요한 태스크를 각 전문 엔진에 배분함.
- 데이터 및 엔진 레이어 (Engine)
    - RAG Engine: VisitKorea API 및 로컬 DB에서 최신 정보를 인출함.
    - Optimization Engine: 외부 지도 API 연동 및 최적화 알고리즘(VRP/TSP)을 통해 타임라인을 산출함.

### **6.2 데이터 모델링 및 AI 구현 방안 (Modeling Strategy)**

단순 LLM의 한계를 극복하기 위해 '검색-추론-계산'이 분리된 하이브리드 모델링을 적용함.

**① 도메인 특화 RAG (Retrieval-Augmented Generation) 모델링**

- 벡터 DB 구축: K-컬처 명소의 스토리텔링, 최신 리뷰, 공공 데이터를 벡터화하여 저장함.
- 동적 그라운딩: 사용자의 질문이 들어오면 구축된 DB에서 유사한 장소를 찾아 실시간으로 장소 관련 정보를 검색하여 LLM에 전달함으로써 할루시네이션을 방지하고 정확도를 극대화함.

**② 멀티모달 인식 및 맥락 연결 (Visual Grounding)**

- Vision-to-Context: 업로드된 이미지를 벡터화 한 후, K-컬처 명소 벡터 DB로 부터 이미지와 유사한 장소를 매칭함.
- 맥락 부여: 이미지와 유사한 장소를 매칭하는 데에서 그치지 않고, 해당 장소와 연결된 K-미디어 콘텐츠 정보를 결합하여 사용자에게 맞춤형 스토리텔링을 생성함.

**③ 추론-최적화 디커플링 (Reasoning-Optimization Decoupling)**

- LLM의 역할: 사용자와 나눈 대화 내용을 바탕으로 사용자의 요구사항(취향, 제약 조건 등)을 파악하고, 이를 구조화된 데이터(JSON) 형태로 변환하는 역할을 수행함.
- 알고리즘의 역할: LLM이 내뱉은 구조화된 데이터를 기반으로 사용자 취향에 맞는 장소를 찾고, 실제 지도 API의 거리/시간 데이터를 사용해 수학적으로 가장 효율적인 경로를 계산함. 이는 LLM이 계산 과정에서 범할 수 있는 오류를 원천 차단하는 핵심 방안임. 장소들 사이 가장 효율적인 경로를 수학적으로 계산하고 LLM이 계산 과정에서 범할 수 있는 오류(국내 교통 현황, 매장 운영 정보 등)를 차단하여 정확도 높은 정보를 제공할 수 있음.

### **6.3 기술 스택**

**① AI & Data Modeling**

- LLM Engine: GPT-4o 또는 Claude 3.5 Sonnet (사용자 의도 파악 및 구조화된 데이터 생성)
- Framework: LangChain 또는 LangGraph (전체 AI 워크플로우 및 멀티 에이전트 설계)
- Embedding & Vector DB: OpenAI Embedding + Pinecone 또는 VectorDB (K-컬처 명소 및 리뷰 데이터 저장 및 RAG 구현)
- Computer Vision: CLIP, DINOv2 또는 GPT-4o 모델 앙상블

**② Optimization & Geo-Data**

- Optimization Engine: Google OR-Tools (VRP/TSP 알고리즘을 활용한 경로 및 타임라인 최적화 연산)
- Maps & Geo API: 네이버 지도 API 또는 카카오 지도 API (국내 도로 네트워크 데이터, 다국어 길 찾기 정보 및 실시간 교통량 확보)
- Data Sources: VisitKorea API (한국관광공사 공공 데이터), Naver 지도 API (장소 상세 및 평점 데이터)

**③ Backend & Infrastructure**

- Language: Python 3.10+ (AI 모델 및 데이터 연산에 최적화)
- Framework: FastAPI (비동기 처리를 통한 빠른 응답 생성 및 LLM 스트리밍 지원)
- Database: RDS(MySQL) + Qdrant(VectorDB)
- Infrastructure: ALB + AWS (EC2)

**④ Frontend**

- Framework: Next.js + React (웹/모바일 대응 및 지도 시각화 인터페이스 구축)
- State Management: TanStack Query (React Query) (실시간 일정 업데이트 및 비동기 데이터 관리)
